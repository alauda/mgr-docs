# 日志管理

## 日志类型
1. 错误日志：`/var/log/mysql/error.log`
2. 慢查询日志：`/var/log/mysql/slow.log` 
3. 审计日志：`/var/log/mysql/audit.log`
4. Group Replication日志：`/var/log/mysql/gr.log`

## 操作步骤
<Tabs>
<Tab label="CLI">
1. 查看实时日志
```bash
kubectl logs -n $namespace $name-primary -f
```

2. 日志归档分析
```bash
kubectl exec -n $namespace $name-primary -- zgrep "ERROR" /var/log/mysql/error.log.1.gz
```
</Tab>

<Tab label="Web Console">
1. 进入 **日志** 页签
2. 选择日志类型：
   - 节点筛选
   - 时间范围选择
   - 关键词过滤
3. 导出日志：
   - 原始日志下载
   - 分析报告生成
4. 关联分析：
   - 错误日志与监控指标关联
   - 慢查询与执行计划分析
</Tab>
</Tabs>
