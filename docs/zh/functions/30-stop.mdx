# 停止/启动实例

## 功能简介
提供实例的临时停止和重新启动功能，用于资源调度维护、成本节约等场景。停止实例将保留所有配置和数据但释放计算资源。

## 前提条件
停止实例要求实例为 **处理中**、**运行中**、**异常** 或 **未知** 状态。

启动实例要求实例为 **停止中** 或 **已停止** 状态。

## 注意事项
为了保障数据安全, 建议在停止实例前先做一次全量备份。
1. 停止实例将终止所有数据库连接
2. 包含 MySQL 数据的 PVC 会保留
3. 重启后需验证 MGR 拓扑完整性

## 操作步骤
<Tabs>
    <Tab label="CLI">
        #### 停止实例
        ```bash
        kubectl patch mysql ${name} -n ${namespace} --type='merge' -p '{"spec":{"pause":true}}'
        ```

        #### 启动实例
        ```bash
        kubectl patch mysql ${name} -n ${namespace} --type='merge' -p '{"spec":{"pause":false}}'
        ```
    </Tab>

    <Tab label="Web Console">
        1. 在左侧导航栏选择 **MySQL-MGR**
        2. 进入目标命名空间
        3. 选择目标实例
        #### 停止实例
        1. 点击 **操作 > 停止**
        2. 确认操作并等待状态变为「已停止」

        #### 启动实例
        1. 在已停止实例的操作菜单中点击 **启动**
        2. 等待状态变为「运行中」
    </Tab>
</Tabs>
