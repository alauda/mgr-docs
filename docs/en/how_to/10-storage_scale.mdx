# Online Expansion

## Use Case

As the volume of data increases, it is important to expand storage in a timely manner before reaching the storage quota, to ensure the continuous availability of the business. The online expansion process will not affect the normal provision of services.

## Steps

<Tabs>
  <Tab label="CLI">
    1. Use the following command to expand the storage capacity:

    ```bash
    kubectl patch mysql -n ${namespace} ${instance_name} --type='json' -p '[{"op":"replace","path":"/spec/mgr/volumeClaimTemplate/spec/resources/requests/storage","value":"${new_storage}"}]'
    ```

    2. Use the following command to check whether the storage capacity expansion was successful:

    ```bash
    kubectl get pvc -n ${namespace} -l "v1alpha1.mysql.middleware.alauda.io/cluster=${instance_name}"
    ```
  </Tab>

  <Tab label="Web Console">
    1. In the left navigation bar, click **MySQL-MGR**.

    2. Click on ***the name of the namespace***.

    3. Click on ***the instance name***.

    4. Under the **Details** tab, click **Update Specification**.

    5. In **Storage Expansion**, modify **New Capacity** to specify the desired increased volume.

    6. Click **Update**.

       After a moment, you will see the **Storage Capacity** increase in the list of persistent volume claims.
  </Tab>
</Tabs>
