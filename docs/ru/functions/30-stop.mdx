# Остановка и запуск экземпляра

## Обзор функции

Предоставляет функции временной остановки и перезапуска экземпляров, применимые для планирования ресурсов, обслуживания, экономии затрат и других сценариев. Остановка экземпляра сохранит все конфигурации и данные, но освободит вычислительные ресурсы.

## Предварительные условия

Остановка экземпляра требует, чтобы экземпляр находился в статусе **Обработка**, **Работающий**, **Аномальный** или **Неизвестный**.

Запуск экземпляра требует, чтобы он находился в статусе **Остановка** или **Остановленный**.

## Примечания

Для обеспечения безопасности данных рекомендуется выполнить полное резервное копирование перед остановкой экземпляра.

1. Остановка экземпляра приведет к завершению всех подключений к базе данных.
2. PVC, содержащий данные MySQL, будет сохранен.
3. После перезапуска необходимо проверить целостность топологии MGR.

## Процедура

<Tabs>
  <Tab label="CLI">
    #### Остановка экземпляра

    ```bash
    kubectl patch mysql ${name} -n ${namespace} --type='merge' -p '{"spec":{"pause":true}}'
    ```

    #### Запуск экземпляра

    ```bash
    kubectl patch mysql ${name} -n ${namespace} --type='merge' -p '{"spec":{"pause":false}}'
    ```
  </Tab>

  <Tab label="Веб-консоль">
    1. Выберите **MySQL-MGR** в левом навигационном баре.
    2. Введите целевое пространство имен.
    3. Выберите целевой экземпляр.

    #### Остановка экземпляра

    1. Нажмите **Действия > Остановить**.
    2. Подтвердите операцию и дождитесь изменения статуса на "Остановлен".

    #### Запуск экземпляра

    1. В меню операций остановленного экземпляра нажмите **Запустить**.
    2. Дождитесь изменения статуса на "Работающий".
  </Tab>
</Tabs>
