# Конфигурация параметров

## Обзор

Настройка соответствующих параметров MySQL в соответствии с бизнес-сценариями может оптимизировать производительность базы данных, лучше поддерживая бизнес-операции. В экземпляре MySQL-MGR по умолчанию используются системные параметры, предоставляемые MySQL и Percona. Вы можете удобно настраивать параметры через интерфейс.

Платформа предоставляет шаблоны параметров для определения набора желаемых параметров MySQL. При создании экземпляра вы можете использовать шаблоны параметров для пакетного изменения параметров по умолчанию, а параметры, не включенные в шаблон, сохранят свои значения по умолчанию.
После того, как экземпляр работает нормально, параметры MySQL могут быть обновлены.

Для получения дополнительной информации о параметрах, пожалуйста, обратитесь к [Официальной документации MySQL](https://dev.mysql.com/doc/refman/8.0/en/innodb-parameters.html) и [Официальной документации Percona](https://docs.percona.com/percona-server/8.0/).

## Примечания

Изменение конфигурации параметров приведет к последовательному перезапуску Pod'ов, что может вызвать кратковременные перерывы в бизнес-операциях.

## Процедура

<Tabs>
  <Tab label="CLI">
    ```bash
    kubectl patch mysql -n ${namespace} ${name} --type='merge' --patch '
    spec:
      params:
        mysql:
          mysqld:
            max_connections: "1000"
            binlog_expire_logs_seconds: "604800"
    '
    ```

    Вы можете изменить параметры, настроив spec.params.mysql в CR, при этом соответствующие ключи параметров должны соответствовать тем, что используются в MySQL. Как показано выше, установка max_connections в 1000 и binlog_expire_logs_seconds в 604800, что равно 7 дням.
  </Tab>

  <Tab label="Web Console">
    #### Настройка параметров с использованием рекомендуемых шаблонов

    **Совет**: Пожалуйста, обратитесь к [Шаблонам параметров] для создания шаблонов параметров.

    - Платформа предоставляет различные шаблоны параметров для любого экземпляра MySQL-MGR в кластере. Каждая спецификация шаблона параметров настроена с системными параметрами, такими как `loose_group_replication_paxos_single_leader: ON` и `sql_generate_invisible_primary_key: ON`, для повышения удобства использования MySQL.

    - Экземпляр MySQL-MGR по умолчанию применяет шаблон параметров с именем **ssd-2c4g-mysql-8.0-mgr**. При создании экземпляра MySQL-MGR вы можете выбрать соответствующий шаблон параметров на основе вашего носителя хранения и спецификаций экземпляра.

    #### Обновление параметров

    После обновления существующей конфигурации параметров для экземпляра, экземпляр автоматически перезапустит реплики MySQL последовательно для применения изменений.

    ##### Процедура

    1. Нажмите **MySQL-MGR** в левой панели навигации.

    2. Нажмите ***Имя пространства имен***.

    3. Нажмите ***Имя экземпляра***.

    4. На вкладке **Конфигурация параметров** нажмите **Обновить параметры**.

    5. Настройте желаемые параметры, индивидуально или через массовый импорт.

       **Примечание**: Убедитесь, что импортируемые параметры не включают неподдерживаемые параметры MySQL, так как это может помешать запуску экземпляра.

    6. Нажмите **Обновить**.

    ##### Связанные операции

    Если вы хотите повторно использовать конфигурацию параметров другого экземпляра, вы можете нажать для экспорта параметров на вкладке **Конфигурация параметров** исходного экземпляра и обратиться к шаблону параметров для создания пользовательского шаблона для массового импорта параметров.
  </Tab>
</Tabs>
