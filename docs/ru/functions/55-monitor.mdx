# Мониторинг

## Обзор функций

Интегрированные в панель данные мониторинга могут быть использованы для мониторинга MySQL и оповещений с точки зрения ресурсов, производительности и емкости, а также поддерживают настройку стратегий уведомлений.

Интуитивно представленные данные мониторинга могут обеспечить поддержку принятия решений для инспекции операций или оптимизации производительности.

### Панель мониторинга производительности

Платформа по умолчанию собирает общие индикаторы мониторинга, относящиеся к ресурсам, производительности и емкости MySQL. На вкладке **Мониторинг** экземпляра можно просмотреть данные индикаторов в реальном времени.

| Индикатор                      | Стандартные индикаторы мониторинга                                                                                                                                                                                                                       |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Использование CPU пода         | Использование CPU пода. Поды с использованием CPU, превышающим 100%, будут игнорироваться, так как в нормальных условиях использование CPU не должно превышать 100%. Если оно превышает 100%, пожалуйста, проверьте, установлены ли все контейнеры пода ограничения ресурсов.      |
| Использование памяти пода      | Использование памяти пода. Поды с использованием памяти, превышающим 100%, будут игнорироваться, так как в нормальных условиях использование памяти не должно превышать 100%. Если оно превышает 100%, пожалуйста, проверьте, установлены ли все контейнеры пода ограничения ресурсов. |
| Сетевой вход                  | Среднее количество байт в секунду, полученное из сети за последние 5 минут                                                                                                                                                                               |
| Сетевой выход                 | Среднее количество байт в секунду, отправленных в сеть за последние 5 минут                                                                                                                                                                               |
| Использовано PVC              | Использование хранилища PVC.                                                                                                                                                                                                                                |
| Соединения MySQL              | Max Connections – это максимальное разрешенное количество одновременных клиентских соединений. По умолчанию это 151. Увеличение этого значения увеличивает количество дескрипторов файлов, необходимых для mysqld.                                               |
| Активность потоков клиента MySQL| Threads Connected – это количество открытых соединений, а Threads Running – это количество потоков, которые не находятся в состоянии ожидания.                                                                                                          |
| Блокировки таблиц MySQL       | MySQL использует различные блокировки по разным причинам. На этом графике видно, сколько блокировок на уровне таблицы было запрошено MySQL у движка хранения.                                                                                                 |
| Сортировки MySQL              | Этот график также показывает, когда сортировки должны были просканировать всю таблицу или определенный диапазон таблицы, чтобы вернуть результаты, которые не могли быть отсортированы с помощью индекса.                                                |
| Медленные запросы MySQL       | Медленные запросы определяются как запросы, которые занимают больше времени, чем значение настройки long\_query\_time.                                                                                                                                  |
| Запросы буфера InnoDB         | Запросы буфера InnoDB относятся к общему количеству запросов на чтение и запись, сделанных в буферный пул InnoDB.                                                                                                                                       |
| TPS                           | Transactions Per Second (TPS) – это ключевая метрика производительности, измеряющая количество транзакций, которые MySQL обрабатывает в секунду.                                                                                                            |
| Команды                       | Команды относятся к количеству SQL-инструкций, выполненных MySQL, включая запросы, вставки, обновления, удаления и другие операции с базами данных.                                                                                                      |
