# Журнал

## Введение в функции

Понимание журналов в процессе работы экземпляра на уровне контейнера, таких как журналы ошибок MySQL. Правильное использование журналов может помочь вам быстро локализовать проблемы и обрабатывать неисправности и исключения.

## Процедура

<Tabs>
  <Tab label="CLI">
    1. Просмотреть все связанные поды экземпляра, включая под mysql и под mysql-router.

    ```bash
    kubectl get pod -n ${namespace} -l "middleware.instance/name=${instance_name}"
    ```

    2. Запросить журналы для конкретного пода.

    ```bash
    kubectl logs -n ${namespace} ${pod_name}
    ```
  </Tab>

  <Tab label="Web Console">
    1. В левой панели навигации нажмите **MySQL-MGR**.

    2. Нажмите ***Имя пространства имен***.

    3. Нажмите ***Имя экземпляра***.

    4. На вкладке **Живой лог** установите параметр просмотра, чтобы увидеть соответствующий журнал.
  </Tab>
</Tabs>
